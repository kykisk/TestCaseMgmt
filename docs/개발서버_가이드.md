# 🛠 개발 서버 가이드

## 🎯 개발 서버 vs 프로덕션 서버

| 항목 | 개발 서버 | 프로덕션 서버 |
|------|----------|--------------|
| **포트** | 5173 | 8090 |
| **시작** | start-dev.bat | start-server.bat |
| **속도** | 빠른 시작 | 최적화된 로딩 |
| **HMR** | ✅ 자동 새로고침 | ❌ 수동 새로고침 |
| **소스맵** | ✅ 있음 (디버깅 쉬움) | ❌ 없음 |
| **외부 접속** | 어려움 | 쉬움 |
| **용도** | 코드 개발/수정 | 실제 사용/데모 |

---

## 🚀 개발 서버 시작

### **방법 1: 배치 파일** (가장 쉬움)
```
TestCase 폴더에서:
start-dev.bat 더블클릭
```

### **방법 2: npm 명령어**
```bash
cd TestCase
npm run dev
```

## 🛑 개발 서버 중지

### **방법 1: 배치 파일**
```
stop-dev.bat 더블클릭
```

### **방법 2: 터미널에서**
```
Ctrl + C
```

---

## ✨ 개발 서버의 장점

### 1. **Hot Module Replacement (HMR)**
```
코드 수정 → 저장 → 자동 새로고침
브라우저 새로고침 불필요!
```

### 2. **빠른 시작**
```
개발 서버: 즉시 시작
프로덕션: 빌드 필요 (5초)
```

### 3. **디버깅 용이**
```
- 소스맵 제공
- 에러 메시지 상세
- React DevTools 사용 가능
```

### 4. **실시간 코드 수정**
```
App.jsx 수정 → 저장 → 화면 즉시 업데이트
```

## 🔄 개발 → 프로덕션 배포

### 개발 중:
```bash
# 개발 서버 실행
start-dev.bat

# 코드 수정
# 테스트
# 디버깅
```

### 배포할 때:
```bash
# 1. 개발 서버 중지
stop-dev.bat

# 2. 프로덕션 빌드
npm run build

# 3. 프로덕션 서버 시작
start-server.bat
```

```
TestCase/
├── start-dev.bat          ✅ 개발 서버 시작
├── stop-dev.bat           ✅ 개발 서버 중지
├── start-server.bat       ✅ 프로덕션 서버 시작
├── stop-server.bat        ✅ 프로덕션 서버 중지
└── package.json           ✅ 스크립트 정의
```

## 💡 사용 시나리오

### 시나리오 1: 새 기능 개발
```
1. start-dev.bat 실행
2. 코드 수정 (src/components/...)
3. 브라우저에서 즉시 확인
4. 계속 개발...
5. stop-dev.bat (완료 후)
```

### 시나리오 2: 버그 수정
```
1. start-dev.bat 실행
2. 브라우저 F12 → DevTools 열기
3. 에러 확인
4. 코드 수정
5. 자동 새로고침으로 확인
```

### 시나리오 3: 데모/시연
```
1. npm run build (최신 빌드)
2. start-server.bat 실행
3. 외부 접속 가능
```

## 🚨 주의사항

### 개발 서버 사용 시:
- ⚠️ 외부 접속 불가 (localhost만)
- ⚠️ 성능 최적화 안 됨
- ⚠️ 디버그 정보 포함

### 프로덕션 서버 사용 시:
- ✅ 외부 접속 가능
- ✅ 최적화됨 (gzip 압축)
- ✅ AWS 배포 가능
