# 엑셀 템플릿 사용 가이드

## 📋 생성된 템플릿 파일

### 1. 요구사항 템플릿
- **파일명**: `requirement_template.xlsx`
- **위치**: `TestCase/public/requirement_template.xlsx`
- **포함 데이터**: 7개의 샘플 요구사항

### 2. 테스트케이스 템플릿
- **파일명**: `testcase_template.xlsx`
- **위치**: `TestCase/public/testcase_template.xlsx`
- **포함 데이터**: 3개의 샘플 테스트케이스

---

## 🎯 요구사항 템플릿 사용법

### 📊 엑셀 컬럼 구조

| 컬럼명 | 필수 | 설명 | 예시 |
|--------|------|------|------|
| 요구사항 ID | ✅ | REQ-YYYYMMDD-XXX 형식 | REQ-20250121-001 |
| 구분 | ✅ | 5가지 중 선택 | 로드맵 체계관리 |
| 기능 요구사항 | ✅ | 구분에 따라 선택 | Hierarchy |
| 설명 | ✅ | 요구사항 상세 설명 | 로드맵 계층 구조... |
| 비고 | ⭕ | 추가 메모 | 3단계 계층까지 지원 |

### 📝 구분 값 (정확히 입력!)

1. **로드맵 체계관리**
   - Hierarchy
   - 구성요소관리
   - 연계관리

2. **로드맵 운영관리**
   - 현황관리
   - Diagram
   - 아이템 리스트
   - 변경점분석
   - 마감
   - 발행
   - Report

3. **관리자 기능**
   - 기준정보 관리
   - 결재관리
   - 공지사항/게시판 관리

4. **인터페이스**
   - IAM
   - SSO
   - Knox결재
   - Knox메일

5. **기타**
   - 정보검색
   - 즐겨찾기
   - 도움말/FAQ

### ⚠️ 주의사항

- **구분**과 **기능 요구사항**은 정확히 매칭되어야 합니다
- 예: 구분이 "로드맵 체계관리"인 경우, 기능 요구사항은 "Hierarchy", "구성요소관리", "연계관리" 중 하나
- 오타가 있으면 검증 오류 발생

## 🎯 테스트케이스 템플릿 사용법

### 📊 엑셀 컬럼 구조

| 컬럼명 | 필수 | 설명 | 예시 |
|--------|------|------|------|
| TC ID | ⭕ | 비워두면 자동 생성 | TC-20250121-001 |
| 제목 | ✅ | 테스트케이스 명 | 로그인 기능 테스트 |
| 설명 | ⭕ | 테스트 상세 설명 | 사용자 로그인... |
| 요구사항 ID | ⭕ | 쉼표로 구분 | REQ-001, REQ-002 |
| 우선순위 | ✅ | High/Medium/Low | High |
| 카테고리 | ✅ | 테스트 유형 | 기능 테스트 |
| 사전조건 | ⭕ | 실행 전 조건 | 회원가입 완료 |
| 스텝1-동작 | ✅ | 첫 번째 스텝 동작 | 아이디 입력 |
| 스텝1-예상결과 | ✅ | 첫 번째 스텝 결과 | 아이디 입력 가능 |
| 스텝2-동작 | ⭕ | 두 번째 스텝 동작 | 비밀번호 입력 |
| 스텝2-예상결과 | ⭕ | 두 번째 스텝 결과 | 비밀번호 입력 가능 |
| 스텝N-동작 | ⭕ | N번째 스텝 (계속 추가 가능) | ... |
| 스텝N-예상결과 | ⭕ | N번째 스텝 결과 | ... |
| 사후조건 | ⭕ | 실행 후 상태 | 로그인 상태 유지 |
| 태그 | ⭕ | 쉼표로 구분 | 로그인, 인증 |

### 📝 우선순위 값

- **High**: 긴급/중요
- **Medium**: 보통
- **Low**: 낮음

### 📝 카테고리 값

- 기능 테스트
- 통합 테스트
- UI/UX 테스트
- 성능 테스트
- 보안 테스트
- 회귀 테스트
- 기타

### ⚠️ 주의사항

- 최소 1개의 테스트 스텝(스텝1-동작, 스텝1-예상결과)은 필수
- 스텝은 순서대로 입력 (스텝1, 스텝2, 스텝3...)
- 중간 스텝을 건너뛰면 안 됨
- 우선순위와 카테고리는 정확히 입력

## 💡 템플릿 다운로드 방법

### 방법 1: 애플리케이션에서 다운로드
```
1. 브라우저에서 애플리케이션 실행
2. 요구사항 → "엑셀 가져오기" → "템플릿 다운로드"
   또는
   테스트케이스 → "엑셀 가져오기" → "템플릿 다운로드"
```

### 방법 2: public 폴더에서 직접 복사
```
TestCase/public/requirement_template.xlsx
TestCase/public/testcase_template.xlsx
```

### 방법 3: 브라우저에서 직접 다운로드
```
http://localhost:5173/requirement_template.xlsx
http://localhost:5173/testcase_template.xlsx
```

## 📥 Import 절차

### 1단계: 템플릿 다운로드
```bash
# 또는 파일 탐색기에서
C:\Workspace\MyWork\TestCase\public\requirement_template.xlsx
C:\Workspace\MyWork\TestCase\public\testcase_template.xlsx
```

### 2단계: 템플릿 작성
- 샘플 데이터를 참고하여 작성
- 필수 필드 반드시 입력
- 구분/기능요구사항 매칭 확인

### 3단계: 업로드
```
애플리케이션 → 요구사항/테스트케이스 메뉴
→ "엑셀 가져오기" 버튼
→ 파일 선택 또는 드래그 앤 드롭
→ 검증 결과 확인
→ "가져오기" 버튼 클릭
```
