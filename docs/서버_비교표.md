# 🔄 개발 서버 vs 운영 서버 비교

## 📊 전체 비교표

| 항목 | 개발 서버 | 운영 서버 |
|------|----------|----------|
| **시작 방법** | `start-dev.bat` | `start-server.bat` |
| **포트** | 5173 | 8090 |
| **DB Prefix** | `dev_` | `prod_` |
| **화면 표시** | 제목 옆 **[DEV]** 노란색 배지 | 배지 없음 |
| **접속 범위** | localhost만 | 외부 접속 가능 |
| **HMR** | ✅ 자동 새로고침 | ❌ 없음 |
| **빌드** | 불필요 | 필요 (`npm run build`) |
| **속도** | 빠른 시작 | 최적화된 로딩 |
| **소스맵** | ✅ 있음 | ❌ 없음 (압축됨) |
| **용도** | 개발/디버깅 | 배포/운영 |

---

## 🗄️ DB 분리

### localStorage 키 구조:

#### 개발 서버 (포트 5173)
```
dev_projects:list
dev_testcases:project-abc
dev_requirements:project-abc
```

#### 운영 서버 (포트 8090)
```
prod_projects:list
prod_testcases:project-xyz
prod_requirements:project-xyz
```

**완전히 독립적!** 개발 데이터와 운영 데이터가 섞이지 않습니다.

---

## 🚀 사용 시나리오

### 시나리오 1: 새 기능 개발
```
1. start-dev.bat 실행
2. 개발 DB에서 테스트 데이터 생성
3. 기능 개발 및 테스트
4. 코드 수정 시 자동 새로고침
5. 완료!
```

### 시나리오 2: 운영 배포
```
1. npm run build (프로덕션 빌드)
2. start-server.bat 실행
3. 운영 DB는 비어있음 (깨끗한 상태)
4. 실제 데이터 입력 또는 엑셀 Import
5. 외부 공개
```

### 시나리오 3: 버그 수정
```
개발:
  start-dev.bat → 버그 재현 → 수정 → 테스트

운영:
  npm run build → start-server.bat → 확인 → 배포
```

## 🎯 접속 주소

### 개발 서버
```
http://localhost:5173
```
**특징:**
- 개발자만 접속 (localhost)
- 화면에 **[DEV]** 표시
- 개발 DB 사용

### 운영 서버
```
로컬: http://localhost:8090
내부망: http://172.31.47.228:8090
외부(AWS): http://[퍼블릭IP]:8090
```
**특징:**
- 외부 접속 가능
- **[DEV]** 표시 없음
- 운영 DB 사용

## 🔍 DB 확인 방법

### 브라우저 F12 → Console:
```javascript
// 개발 DB 확인
Object.keys(localStorage).filter(k => k.startsWith('dev_'))

// 운영 DB 확인
Object.keys(localStorage).filter(k => k.startsWith('prod_'))

// 현재 환경 확인
console.log(import.meta.env.DEV ? 'DEV 모드' : 'PROD 모드')
```

## 🎉 요약

### ✅ 완료된 작업
1. **DB 완전 분리**
   - 개발 DB: `dev_` prefix
   - 운영 DB: `prod_` prefix

2. **화면 구분**
   - 개발: 제목 옆 **[DEV]** 노란색 배지
   - 운영: 배지 없음

3. **서버 스크립트**
   - 개발: `start-dev.bat` (포트 5173)
   - 운영: `start-server.bat` (포트 8090)

```bash

start-dev.bat → http://localhost:5173 [DEV]

```
